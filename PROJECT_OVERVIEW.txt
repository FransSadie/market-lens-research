PROJECT OVERVIEW (LIVING DOCUMENT)
Last updated: 2026-03-30 (market research pivot)

==================================================
TECHNICAL TERMS
==================================================

1) Purpose
- Build a market research and signal-discovery tool.
- The active product is no longer centered on model training.
- The main goal is to surface useful market context quickly:
  market regime, sector leadership, relative strength, and scan candidates.

2) Active Architecture
- Backend: FastAPI
- Frontend: React + Vite + Tailwind
- DB access: SQLAlchemy ORM
- DB target: PostgreSQL (SQLite fallback supported)
- Scheduling: APScheduler
- Active data source: Yahoo Finance via yfinance
- Active analytics style: tabular price analytics and ranking logic

3) Active Data Flow
- Ingestion layer:
  - Yahoo OHLCV rows -> market_prices
  - every job run logged -> ingestion_runs
- Analytics layer:
  - rolling per-ticker calculations -> analytics_snapshots
  - active use is as analytics snapshots, not primarily ML inputs
  - analytics include:
    multi-horizon returns, RSI, Bollinger z-score, breakout, drawdown,
    trend slopes, volume regime features, and relative-strength features against SPY, QQQ, and sector ETFs
- Research layer:
  - market overview from macro and sector ETFs
  - relative-strength leaders and laggards
  - setup scans: breakout, pullback, volatility expansion
  - daily summary text assembled from current market state
- Observability layer:
  - /data/status and /data/quality endpoints
- UI layer:
  - one-click dashboard for overview, rankings, scans, ops, quality, and docs

4) Active Tables
- market_prices
  - OHLCV rows per ticker/timestamp/interval
- analytics_snapshots
  - analytics snapshot rows built from price behavior
  - includes price, trend, volatility, volume, and relative-strength fields
- ingestion_runs
  - pipeline audit log

Secondary / legacy tables still may exist in the codebase:
- market_labels
- model_runs
- prediction_logs
- older news-related tables from previous experiments

Those are no longer the center of the product.

5) Active Modules
- app/ingestion/market_data.py
  - market price download + normalization
- app/ingestion/pipeline.py
  - active market ingestion flow
- app/analytics/pipeline.py
  - analytics snapshot generation
- app/analytics/market.py
  - market overview, rankings, scans, and summary assembly
- app/api/routes.py
  - API endpoints for research workflow plus operational endpoints
- frontend/src/App.jsx
  - research-desk dashboard

6) Active API Endpoints
- GET /health
- POST /ingest/run
- GET /ingest/status
- POST /pipeline/run
- GET /pipeline/status
- GET /research/overview
- GET /research/rankings?limit=25
- GET /research/scans
- GET /research/summary
- GET /data/status
- GET /data/quality
- POST /run/full
- GET /docs/text

7) Current State
- The repo now behaves like a market research product.
- The frontend is organized around:
  - overview
  - rankings
  - scans
  - operations
  - data quality
  - docs
- Research outputs are generated from the latest analytics snapshots in the database.
- The app can already surface useful daily market context from the existing price pipeline.

8) Legacy / Transitional Code
- Model training, inference, experiment matrix, and older news work may still exist in the repository.
- They are transitional baggage from the previous repo direction.
- They are not the primary workflow of this repo anymore.

9) Near-Term Product Direction
- keep improving market overview quality
- expand relative-strength scoring
- add market breadth and sector rotation views
- add watchlists, exports, and alerts
- continue reducing model-first language and flows from the product
